# Instruções do Projeto Portfolio

## Antes de Começar
1. Instale dependências: `npm install`
2. Inicie dev: `npm run dev`
3. Acesse: `http://localhost:5173`

## Padrão de Commits
- `feat: descrição` - nova funcionalidade
- `fix: descrição` - correção de bug
- `docs: descrição` - documentação
- `style: descrição` - formatação
- `refactor: descrição` - refatoração
- `chore: descrição` - tarefas auxiliares

## ⚠️ IMPORTANTE - ATUALIZAÇÃO DE PORTFÓLIO
Após CADA commit significativo (não inclua `style:`, `chore:` menores):
1. Edite `.project.md` e atualize seção "Última Atualização"
2. Atualize o % de "Progresso" se houver mudança de escopo
3. Faça o commit do projeto normalmente
4. Rode `python AgenteMestre/sincronizar_portfolio.py` para atualizar o Portfolio

- Sempre fazer branch antes de mudanças grandes

- Atualizar `src/data/projetos.json` com dados sincronizados

## Páginas Individuais de Projeto

Cada projeto tem uma rota dedicada para apresentação profissional. O roteamento usa **react-router-dom v7**.

### Rotas existentes
| Projeto | Rota | Arquivo |
|---|---|---|
| Sistema de Gestão de Tempo | `/projetos/sistema-gestao-tempo` | `src/pages/projects/SistemaGestaoTempo.tsx` |
| FungiFresh | `/projetos/fungifresh` | `src/pages/projects/FungiFresh.tsx` |
| Vendinha | `/projetos/vendinha` | `src/pages/projects/Vendinha.tsx` |
| AtaFácil | `/projetos/atafacil` | `src/pages/projects/AtaFacil.tsx` |

### Como adicionar um novo projeto com página própria
1. Criar `src/pages/projects/[NomeProjeto].tsx` (scaffold disponível nos arquivos existentes como referência)
2. Adicionar a rota em `src/App.tsx`:
   ```tsx
   <Route path="/projetos/[slug]" element={<NomeProjetoPage />} />
   ```
3. Adicionar `projectPath: '/projetos/[slug]'` nos dados do projeto em `SimpleApp.tsx` dentro do array `projects`

### Estrutura de rota de projeto
- `← PORTFOLIO` link no topo (usa `<Link to="/">` do react-router-dom)
- Cada página é totalmente independente — pode ter seu próprio design, seções, cores
- A página é para **apresentação profissional** do projeto: problema, solução, stack, prints, resultados
